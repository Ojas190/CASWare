# Backend 

1. Python 3.10
2. face_recognition (Dlib)
3. DeepFace (Facenet512)
4. VGG-Face
5. Mediapipe / RetinaScan (For backup incase Module Deepface failed to load)
6. Supervised and Semi-Supervised Machine Learning Techniques
7. TensorFlow
8. CNN Model for Image training
9. SVM Model for CNN Identifying and hanlding excel sheet in backend
10. MySQL 9.0 (Using module pymysql for Databases / Schema)

# Frontend

1. Python Flask with the help of FastAPI
2. HTML5
3. CSS3
4. Javascript for DOM Manipulations
5. Some Tailswind.css Icon maniuplation in the Frontend

# Features

1. Captures Live Image (from the Campus or institute consists) of User from CCTV to take Attendance (and logs of whichit  consist of Faculty, Student, and Higher Authorities)
2. Scalable backend structure and Easy to Handle
3. Dataset (of Users) needs to be updated as Per on ID card on a term of 5 Yearly biases.
4. Simple, Fast and All-in-one Mini-ERP for Students, Colleges and Government.
5. API Endpoints: GET / → Check if backend is running
6. API Endpoints: POST /predict → Upload image and get prediction

# Why Government And College?

2. Validates the need of Scholarship given to those with actual Evidence rather than fragmented physical documents.
3. Aides Parents and Faculty coordination for seeing the Student is Attending Regularly as this Avoids Proxy.
4. Since the Models Uses polymorphying technique to see 3D face, reduces any Proxy and validates Actual Attendance.
5. CAS works as an Distributed system which helps in telling the students(and their Parents) about there progress, Faculty on keeping records of Students Digital transparent (in CASWare) and Filtering Bright and talented ones and aiding them for brighter future.
